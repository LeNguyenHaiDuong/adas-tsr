## Purpose

Bài này tóm tắt cách một feature TSR production thường được kiểm thử và đánh giá trưởng thành.

## Why It Matters for TSR

Một feature perception không thể được đánh giá chỉ bằng:

- một video demo đẹp,
- hoặc vài metric model card.

Cần nhìn theo nhiều mức kiểm thử khác nhau.

## Test Levels

| Level | Ý nghĩa ngắn |
|---|---|
| Unit Test | Kiểm tra hàm/module nhỏ |
| Integration Test | Kiểm tra nhiều module ghép với nhau |
| MIL | Model in the Loop |
| SIL | Software in the Loop |
| HIL | Hardware in the Loop |
| VIL | Vehicle in the Loop |
| Road Test | Chạy xe hoặc replay route thực tế |
| Regression Test | Đảm bảo version mới không làm tụt hành vi cũ |

## Production Implementation Pattern

Một production flow hợp lý thường cần:

1. replay và benchmark ổn định,
2. scenario catalog,
3. KPI theo event/track, không chỉ theo frame,
4. release gate và evidence bundle.

## Relation to Current Prototype

Prototype hiện mạnh nhất ở:

- replay video cục bộ,
- benchmark runtime thô,
- phân tích thủ công bằng overlay.

Còn thiếu:

- machine-readable logs,
- event-level KPI,
- scenario tagging,
- release gate lặp lại được.

## First Practical Steps for This Repo

- thêm `JSONL` output,
- khóa protocol benchmark,
- thêm temporal confirmation lite,
- thêm KPI script tối thiểu.
